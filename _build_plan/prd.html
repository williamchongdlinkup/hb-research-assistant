HB Research Assistant

A focused web app that transforms the Humanistic Buddhism Research Bibliography — a curated corpus of 1,833 scholarly entries maintained by Nan Tien Institute — into a live AI research assistant. The first release delivers two capabilities no existing tool offers for this corpus: a multi-turn grounded Q&A interface where every AI response cites specific bibliography entries, and a reference audit tool where journal editors and researchers paste a bibliography and instantly see what key HB literature is missing. Built for HB scholars, journal editors, and PhD students attending the 12th HB Symposium and beyond.

Python FastAPI SQLite FTS5 Gemini 3.1 Flash Lite Tailwind CSS Vanilla JS Railway

What the app does

  • Landing page

    Introduces the tool to first-time visitors with corpus stats and three clear entry points into the app.

  • Corpus search

    Keyword search over all 1,833 entries with filters for publication year range and type.

  • Grounded Q&A with citations

    Multi-turn research conversations (up to 5 turns) where every AI claim links to a specific corpus entry.

  • Reference audit

    Paste any bibliography and get back verified citations, missing HB literature, and ranked suggestions.

Already provided by the existing corpus CSV

  • 1,833 curated bibliography entries
  • Abstracts for most entries
  • Full author, editor, journal, publisher metadata
  • Source URLs for most entries
  • Publication year range 2000–2025

Out of scope (v1)

  • User accounts / login — no sign-in, no saved sessions, no personal history; the tool is open and stateless
  • Semantic / vector search — keyword + full-text search only; embeddings and vector database come in Phase 2
  • Gap mapping & research question refinement — powerful but needs more AI scaffolding than this sprint allows
  • Literature review generator — Phase 2
  • Teaching / syllabus builder — Phase 2
  • Corpus contribution workflow — researchers cannot submit new entries yet; CSV is the source of truth
  • Citation export (BibTeX, Zotero, APA) — Phase 2
  • Saving conversations or audit results — sessions are browser-only and reset on close
  • Multiple AI models or user-supplied API keys — one model (Gemini 3.1 Flash Lite), one server-side key
  • Admin panel / in-app corpus editing — corpus is read-only; updated via CSV replacement
  • Multi-corpus / Direction B architecture — HB corpus only; open-source engine is a later phase

External integrations

Google Gemini API

AI reasoning

Powers both the Q&A and reference audit features. The app pre-filters corpus entries by keyword, then sends the relevant entries plus the user's question to Gemini 3.1 Flash Lite, which returns a grounded response. Free tier: 1,500 requests/day, 15 requests/minute — sufficient for a symposium demo.

Credentials needed

  • GEMINI_API_KEY — from aistudio.google.com

Milestones

1

Foundation

Sets up the entire project skeleton, loads the CSV corpus into a searchable database, and gets the app live on a public URL. Everything later builds on this.

What gets built

  • FastAPI project structure with CSV → SQLite FTS5 ingestion on startup
  • Keyword search endpoint returning ranked results
  • Result cards showing title, first author, year, type, and abstract excerpt
  • Year range and publication type filters
  • Expandable entry detail (full abstract + URL opens in new tab)
  • Result count display ("Showing X of N entries", N = live count from /api/stats, currently 1,833)
  • Deployed live to Railway with a working public URL

Not in this milestone

  • Gemini API or any AI features
  • Landing page (comes in Milestone 4)
  • Final visual polish

Done when: A researcher can visit the live Railway URL, type "compassion" in the search bar, and see a filtered list of relevant corpus entries with abstracts, all within 1 second.

2

Grounded Q&A

Adds the AI research conversation layer. Researchers ask questions in natural language and receive grounded, cited answers drawn from the HB corpus — not from the open internet.

What gets built

  • Gemini API integration using GEMINI_API_KEY from environment
  • Chat-style interface with text input and conversation thread
  • Multi-turn conversations up to 5 turns, history visible throughout
  • Thinking / loading indicator while response generates
  • AI response in flowing prose with inline superscript citation numbers
  • Numbered reference list below each response, each entry expandable
  • "Based on X entries from the HB corpus" label on each response
  • "Start new conversation" button available at any time; auto-prompted at 5 turns
  • Honest "outside corpus" response when the question can't be grounded

Not in this milestone

  • Reference audit (Milestone 3)
  • Saving or exporting conversations
  • Sources outside the HB corpus

Done when: A researcher can ask "What does HB scholarship say about compassion and social action?" and receive a multi-paragraph prose response with numbered superscript citations, each linking to a verifiable corpus entry.

3

Reference Audit

Adds the journal editor tool. A researcher or editor pastes any bibliography and instantly learns what's verified in the HB corpus, what's missing, and what else is relevant.

What gets built

  • Text area for pasting a bibliography in any citation format
  • "Audit" button that triggers the analysis
  • Three-section results: ✅ Verified / ⚠️ Missing / 💡 Suggested
  • Each result shows title, author, year, and a one-line relevance reason
  • "Copy suggestions as APA" button for the Missing and Suggested sections

Not in this milestone

  • Saving audit results
  • Batch auditing multiple papers
  • Detecting citation formatting errors
  • Sources outside the HB corpus

Done when: A journal editor can paste a paper's bibliography, click Audit, and within 30 seconds see a three-section report identifying which citations are in the HB corpus and which key HB papers are missing.

4

Polish & Launch

Adds the landing page, unifies the visual design across all views, and prepares the app for its debut at the 12th HB Symposium on 20–21 June 2026.

What gets built

  • Landing page: NTI logo, 2–3 sentence description, three entry points, corpus stats, citation request, link to existing visualiser
  • Unified visual design across all pages — clean, professional, academic tone
  • End-to-end testing with 20+ real research queries across all three features
  • Final Railway deploy with stable public URL ready to demo

Not in this milestone

  • New features — scope is frozen for the symposium
  • Mobile app or browser extension
  • Phase 2 features (vector search, gap mapping, etc.)

Done when: A first-time visitor lands on the page, understands what the tool does within 10 seconds, navigates to Q&A, asks a real research question, and gets a grounded cited response — all without instruction.
